## 第一步：解析股票代码与市场

从用户输入中判断市场类型，构造对应数据源 URL：

**A 股**（`.SH` / `.SZ` 后缀或纯6位数字）
- `000001.SZ` → 纯数字 `000001`，使用同花顺 F10

**港股**（`.HK` 后缀或4-5位数字）
- `0700.HK` → 同花顺港股代码 `HK00700`（补足5位），使用同花顺港股 F10 + AAStocks

**美股**（纯字母 ticker）
- `BABA` → 使用 Macrotrends + Stock Analysis + Yahoo Finance

## 第二步：调用 Kimi_Fetch 抓取基本面数据

### A 股：同花顺 F10（9个页面）
F10 URL 模板：`https://basic.10jqka.com.cn/{纯数字代码}/{页面}.html`

| 页面 | 内容 |
|------|------|
| `company.html` | 公司基本信息、主营业务、所属行业、公司简介 |
| `finance.html` | 财务指标：营收、利润、毛利率、ROE、资产负债率 |
| `holder.html` | 股东结构：十大流通股东、机构持仓、股东户数 |
| `worth.html` | 估值分析：PE、PB、PEG、机构预测 EPS |
| `equity.html` | 股本结构：总股本、流通股、限售股 |
| `capital.html` | 资本运作：分红、增发、回购记录 |
| `news.html` | 最新新闻公告、研究报告 |
| `event.html` | 公司大事记、重组并购（可能受限） |
| `filed.html` | 公司章程、资料文件（可能受限） |

> **注意**：`event.html` 和 `filed.html` 部分股票可能访问受限，失败则跳过。

> **编码说明**：同花顺 F10 页面使用 **GBK** 编码，`kimi_fetch` 会自动处理编码转换，无需手动解码。

### 港股：同花顺港股 F10 + AAStocks

代码格式：`HK` + 5位数字（不足5位补零），如 `0700.HK` → `HK00700`

**同花顺港股 F10**（与 A 股页面结构相同，优先使用）：
```
https://basic.10jqka.com.cn/HK{5位代码}/{页面}.html
```
可用页面：`company` / `finance` / `holder` / `worth` / `capital` / `news`

**AAStocks 补充**（同花顺抓取失败时使用）：

- 公司资料：`https://www.aastocks.com/sc/stocks/analysis/company-fundamental/company-profile/{5位代码}/1/1`

**Yahoo Finance 估值**：`https://finance.yahoo.com/quote/{ticker}.HK/financials/`

### 美股：Macrotrends + Stock Analysis

`{slug}` = 公司名小写连字符，如 `BABA` → `alibaba`；先访问 `https://www.macrotrends.net/stocks/charts/{ticker}` 确认实际 slug。

| 资产负债率 | `https://www.macrotrends.net/stocks/charts/{ticker}/{slug}/debt-to-equity-ratio` |

| 公司概况 | `https://stockanalysis.com/stocks/{ticker}/` |

Yahoo Finance 作为补充：`https://finance.yahoo.com/quote/{ticker}/financials/`

## 第三步：从 Markdown 内容中提取关键数据

### company.html → 公司概况
- 公司全称、股票简称、上市时间、注册地址、法人代表、员工人数
- 所属行业（申万一级/二级分类 + 证监会分类）
- 主营业务（完整描述，不要截断）
- 公司简介（控制在 300 字以内，突出核心竞争力）
- 官网、联系方式

### finance.html → 财务指标
- 最新财报期（年报/季报，注明截止日期）
- **收入利润**：营业收入、归母净利润（近 3 年数据 + 同比增速）
- **盈利能力**：毛利率、净利率、ROE、ROIC（近 3 年趋势）
- **偿债能力**：资产负债率、流动比率、速动比率、利息保障倍数
- **运营能力**：应收账款周转率、存货周转率、总资产周转率
- **现金流**：经营活动现金流净额、自由现金流（近 3 年）
- **每股指标**：EPS、每股净资产、每股经营现金流
- 财务诊断关键词（盈利能力评级、成长能力评级、健康度评级）

### holder.html → 股东结构
- 十大流通股东完整列表（名称 + 持股数量 + 持股比例 + 较上期变化）
- 十大股东中机构类型分布（基金/社保/险资/QFII/私募等）
- 机构持仓总比例 + 环比变化
- 股东户数 + 人均持股市值 + 环比变化（增户/减户信号）
- 股权集中度分析（前十大持仓占比）

### worth.html → 估值分析
- PE（TTM）、PE（静态）及行业平均、历史分位
- PB 及行业平均、历史分位
- PEG（基于一致预期增速）
- PS（市销率）
- EV/EBITDA
- 机构关注家数 + 买入/增持/中性/减持评级分布
- 未来 1-3 年一致预期 EPS、营收、净利润

### equity.html → 股本结构
- 总股本、流通 A 股、限售股（数量 + 比例）
- 限售股解禁时间表（如有近期解禁）
- 股权质押情况（质押比例 + 预警线）

### capital.html → 资本运作
- 近 5 年分红记录（每股派息、分红率、股息率）
- 历次增发/配股（时间、规模、价格）
- 股票回购（时间、规模、均价）
- 重大资产重组情况（如有）

### news.html → 最新动态
- 最新 5 条公告（标题 + 日期 + 简要摘要）
- 最新 3 条研究报告（机构 + 评级 + 目标价 + 日期）

### event.html → 重大事项（如能获取）
- 近 1 年重大事项（融资、并购、业务拓展、监管处罚等）

## 第五步：保存文件并发送给用户

### 5.1 保存报告到本地

使用 **Write 工具**将报告保存为文件：

**保存路径**：`~/.openclaw/skills/stock-assistant/data/fundamental/{纯数字代码}/{纯数字代码}_fundamental_{YYYYMMDD}.md`

例如分析 000001，报告日期为 2026-03-14：
```
~/.openclaw/skills/stock-assistant/data/fundamental/000001/000001_fundamental_20260314.md
```

### 5.2 推送报告

读取 `~/.openclaw/skills/stock-assistant/config.json` 中的 `push.channel`，按以下规则处理：

**channel = `openclaw`**
直接将报告内容输出到 stdout，openclaw 会自动捕获推送。

**channel = `feishu`**
将报告内容推送给用户飞书（app_id / app_secret / user_id 均从 config.json 读取）。

**channel = `both`**
先输出到 stdout，再执行飞书推送。

**channel 未配置或读取失败**
默认直接在对话中输出报告内容，并提示用户检查 `push.channel` 配置。

## 执行要点

- **数据缺失**：某页面返回空或 HTTP 错误时，调用其他可用搜索 API
- **数字格式**：金额统一换算为"亿元"或"万元"，保留 1-2 位小数；百分比保留 2 位小数
- **历史数据**：财务指标尽量提取近 3 年数据，体现趋势而非单点数据
- **综合评估**：最后章节基于所有数据给出判断，观点鲜明，不要只堆砌数字
- **不要虚构**：页面抓取失败的数据用"—"或"暂无数据"标注，严禁编造数字
