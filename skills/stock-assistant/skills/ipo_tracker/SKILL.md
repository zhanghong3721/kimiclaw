# 🆕 打新助手

## 角色定位

帮用户跟踪新股申购节奏，核查打新资格，提醒关键时间节点（申购日、缴款日、上市日），并对重点新股给出简要基本面判断——值不值得打、风险在哪。

---

## 执行流程

### Step 1：读取用户持仓

从 `~/.openclaw/skills/stock-assistant/watchlist.json` 读取持仓，用于：
- 估算沪市/深市市值（A股打新资格核查）
- 判断是否持有港股通标的（港股通打新参考）

> 注意：watchlist.json 里的市值只是参考，实际打新资格以用户券商账户的 T-2 日日均市值为准，需提示用户自行确认。

---

### Step 2：获取新股日历（并行）

**A 股新股**
- 工具：`kimi_fetch`，URL：`https://data.eastmoney.com/xg/xg/`
- 获取：近期及本周申购的新股列表，包含：股票名称、代码、板块（主板/科创板/创业板/北交所）、申购日期、申购上限（顶格申购数量）、发行价、募资规模
- 补充：`kimi_search` 搜索 `本周新股申购 {当前日期}` 确认是否有遗漏

**港股通新股**
- 工具：`kimi_search`，搜索 `港股通 新股 申购 {当月}`
- 工具：`kimi_fetch`，URL：`https://data.10jqka.com.cn/datacenterph/ipo/indexPage.html#/hkStock`
- 获取：近期港股 IPO 列表，标注哪些支持港股通申购（非所有港股 IPO 都支持）
- 重点字段：股票名称、招股日期、定价区间、募资规模、所属行业、是否支持港股通

**各新股公司简介（并行搜索）**

对日历中每只新股，用以下方式补充基本面信息：

*A 股新股：*
- `kimi_search`：`{股票名} 招股书 主营业务 财务数据`
- `kimi_fetch`：`http://www.sse.com.cn/disclosure/listedinfo/announcement/`（科创板）或深交所招股书公告页，获取募资用途、核心财务指标（营收/利润/毛利率近三年）
- `kimi_search`：`{股票名} 新股 机构询价 网下认购 倍数`，获取网下认购热度

*港股新股：*
- `kimi_search`：`{股票名} 招股书 主营业务 {当月}`
- `kimi_fetch`：`https://www1.hkexnews.hk/search/titlesearch.xhtml`，搜索该公司招股说明书（Prospectus）
- `kimi_search`：`{股票名} IPO 基石投资者 超额认购`，了解机构参与情况

重点提取：主营业务、近三年营收/利润趋势、行业地位、发行 PE/PS 与同行业二级市场可比公司对比、大股东背景及锁定期。

---

### Step 3：打新资格核查

**A 股打新资格**

| 板块 | 市值要求 | 计算方式 | 申购上限 |
|------|---------|---------|---------|
| 主板（沪市） | 沪市市值 ≥ 1万元 | T-2 日日均，每 1万元可申购 1000股 | 顶格申购上限因股而异 |
| 主板（深市） | 深市市值 ≥ 1万元 | T-2 日日均，每 1万元可申购 500股 | 同上 |
| 科创板 | 沪市市值 ≥ 10万元 + 开通科创板权限 | T-2 日日均 | 最高 1.5万股 |
| 创业板 | 深市市值 ≥ 1万元 + 开通创业板权限 | T-2 日日均 | 最高 3万股 |
| 北交所 | 开通北交所权限 | 无市值要求 | — |

根据用户 watchlist 中的 CN 持仓估算市值，给出参考资格判断，并**提醒用户以券商账户实际显示为准**。

**港股通打新资格**

港股通打新门槛：
- 需开通港股通权限（资产门槛通常 50万元人民币）
- 每手申购金额通常在几千至数万港元不等
- 部分新股仅面向专业投资者，普通投资者不可参与

根据 watchlist 中 HK 市场持仓判断用户是否已开通港股通，如无则提示开通条件。

### Step 4：生成报告

---

#### 一、本周打新日历

按时间顺序列出本周及未来 7 日的新股节点：

```
📅 {日期}（{星期}）
  申购：{股票名}（{代码}）{板块} · 发行价 ¥{price} · 申购上限 {X}股
  缴款：{股票名}（{代码}）· 截止 {时间}
  上市：{股票名}（{代码}）· 发行价 ¥{price}
```

无节点的日期略过不写。

---

#### 二、重点新股解读

对本周申购的新股，每只覆盖以下内容（**可打则详写，建议回避则简写**）：

**{股票名}（{代码}）{板块}**

- **基本情况**：主营业务一句话描述、所属行业、募资用途
- **发行定价**：发行价 ¥X，对应 PE/PS 倍数，与同行业 A 股可比公司估值对比
- **机构认购**：战略配售比例、网下机构认购倍数（反映机构热度）
- **打新价值判断**：
  - ✅ 建议申购：说明理由（行业景气度高、估值合理、机构超额认购倍数高等）
  - ⚠️ 谨慎参与：说明风险（高估值、行业下行、募资超募、大股东减持压力等）
  - ❌ 建议回避：明确说明原因
- **上市首日预判**：参考同板块近期新股首日表现，给出大致涨幅区间预估（仅参考，不构成建议）

---

#### 三、打新资格小结

基于 watchlist 持仓估算，列出用户当前可参与的板块：

```
根据 watchlist 估算（仅供参考，以券商账户为准）：
  沪市持仓市值约 ¥{X}
    → 主板（沪）：✅ 可申购，顶格约 {X}股
    → 科创板：{✅ 可申购 / ❌ 不足10万，差约¥X}

  深市持仓市值约 ¥{X}
    → 主板（深）：✅ 可申购，顶格约 {X}股
    → 创业板：{✅ 已开通 / ❓ 请确认是否开通权限}

  港股通：{✅ 已有港股持仓，应已开通 / ❓ watchlist 无港股，请确认是否开通}
```

---

#### 四、待缴款 & 中签追踪

提醒用户近期需要操作的事项：

- **待缴款**：列出已过申购期、等待中签结果或需缴款的新股，注明缴款截止时间
- **待上市**：已缴款、等待上市的新股，标注预计上市日期
- **中签查询方式**：提示用户在券商 App「新股」→「中签查询」确认，或搜索 `{股票名} 中签结果`

---

### Step 5：推送报告

读取 `~/.openclaw/skills/stock-assistant/config.json` 中的 `push.channel`，按以下规则处理：

**channel = `openclaw`**
直接将报告内容输出到 stdout，openclaw 会自动捕获推送。

**channel = `feishu`**
调用飞书 API 将报告推送给用户（app_id / app_secret / user_id 均从 config.json 读取）。

**channel = `both`**
先输出到 stdout，再执行飞书推送。

---

## 边界情况

| 情况 | 处理 |
|------|------|
| 本周无新股申购 | 输出"本周暂无新股申购，下期新股日历见下方"，列出未来2周的节点 |
| watchlist 无持仓市值 | 跳过资格估算，提示用户手动告知市值 |
| 港股 IPO 不支持港股通 | 明确标注"仅限香港本地账户，港股通不可申购" |
| 用户询问某只具体新股 | 直接执行 Step 4 的重点解读，跳过日历部分 |
| 用户询问是否中签 | 引导用户去券商 App 查询，同时 `kimi_search` 搜索该股中签率供参考 |

---

⚠️ *打新存在风险，破发情况时有发生。本报告仅供参考，不构成投资建议。申购前请以券商账户实际资格为准。*
